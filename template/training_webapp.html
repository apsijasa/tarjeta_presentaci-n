SwimX Training

A Comprehensive Swimming Club Management System

Dashboard Preview Image
Volver al Portafolio

Project Overview

SwimX Training is a full-featured web application designed to streamline the management of swimming clubs. The system incorporates multiple roles, training programs, attendance tracking, and comprehensive analytics for swim coaches and administrators.

Technologies Used

  • Frontend: Angular 8.0.6
  • Backend: PHP 7.4.0 with Laravel
  • Database: MySQL 8.0.27
  • Server: Ubuntu 22.04 with Apache 2
  • Storage: S3 Bucket (Digital Ocean Spaces)

Key Features

User Role Management

Multiple user roles including Administrator, Club Manager, Coach, Cashier, and Student, each with specific permissions and access levels.

Training Program Creation

Comprehensive tools for creating, editing, and managing swimming training programs with specific sets, techniques, and progressions.

Attendance Tracking

Detailed attendance tracking for both club access and specific training sessions with reporting capabilities.

Performance Analytics

Dashboards for individual swimmers and groups with analytics on progress, training completion, and performance metrics.

Payment Processing

Integrated payment system for managing club memberships, training fees, and workshop registration.

Annual Planning

Advanced planning tools for creating annual training plans with periodization, competition planning, and detailed microcycles.

System Architecture

SwimX Training utilizes a client-server architecture offering flexibility and scalability to support multiple platforms from a central system. The application is developed with two different frameworks:

Architecture Diagram

Backend (Laravel)

The Laravel-based backend provides RESTful API services with token-based authentication. It manages data persistence, business logic, and integrates with external services for storage and notifications.

Key configurations include database connections, email service integration, and S3 bucket storage settings.

Frontend (Angular)

The Angular frontend delivers a responsive user interface that communicates with the backend through API calls. The application features role-specific dashboards and modules for different user types.

Environment configurations handle API endpoint connectivity and base URL settings.

System Modules

Student Management

Register, view, and edit student profiles with personal information, emergency contacts, and swimming metrics.

Training Management

Create and manage swimming workouts with specific sets, exercises, and training parameters.

Workshop Management

Organize workshops with registration, attendance tracking, and group assignment features.

Catalog Management

Maintain catalogs for swimming techniques, equipment, and training methodologies.

Payment Processing

Record, track, and manage student payments with different payment methods and receipt generation.

Attendance Tracking

Track student attendance for both club access and specific training sessions.

Human Resources

Manage staff accounts and permissions for various system roles.

Dashboard & Analytics

Visual representation of training data, attendance statistics, and student performance metrics.

Implementation & Challenges

User Role Management

Implemented a hierarchical permission system to ensure users could only access modules and data relevant to their role (Administrator, Manager, Coach, Cashier, Student).

Training Planning

Developed a complex training planning system that allows coaches to create detailed workout plans with tracking for stroke types, energy zones, and training parameters.

Annual Planning

Created a visual calendar-based interface for annual training plans with periodization, competition periods, and advanced metrics tracking.

API Architecture

Designed a comprehensive API with token-based authentication to ensure secure communication between frontend and backend components.

Results & Learnings

SwimX Training provides swimming clubs with a robust management solution that streamlines administrative tasks, enhances training program development, and offers valuable insights through analytics.

Key Takeaways

  • Building complex role-based systems requires careful planning of permissions and access controls
  • Client-server architecture provides flexibility for future expansion to mobile platforms
  • Dashboard visualizations help coaches and administrators quickly interpret performance data
  • Configuration files simplify deployment and environment management
